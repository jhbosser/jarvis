# Report Layouts

This repository currently produces console-oriented outputs and file-based persisted records rather than formal spreadsheet reports. Even so, output structures must remain stable.

Reports and outputs must not change without updating this file.

## Current Output Layouts

### CLI Talk Output
- Report name: `talk`
- Format: plain text console status
- Expected lines:
  - capture confirmation
  - assistant response
  - conversation file path
  - summary file path
- Important notes:
  - the command must also persist conversation and summary files

### CLI Search Output
- Report name: `search`
- Format: console table plus optional memory map blocks
- Expected columns:
  - `#`
  - `Entry`
- Column order:
  - `#`
  - `Entry`
- Important notes:
  - table appears only when log matches exist
  - log matches include active and archived memory log slices
  - memory map matches are shown as plain text blocks after the table

### CLI Today Output
- Report name: `today`
- Format: console table
- Expected columns:
  - `#`
  - `Entry`
- Column order:
  - `#`
  - `Entry`
- Important notes:
  - title includes the current date
  - each row mirrors a single line from active or same-day archived log slices

### Saved Log Record Layout
- Report name: `memory_log`
- Format: plain text line
- Expected columns:
  - timestamp
  - category
  - description
- Column order:
  - timestamp
  - category
  - description
- Important notes:
  - separator is ` - `
  - layout must remain compatible with `utils._parse_log_line()`

### Conversation Log File Layout
- Report name: `conversation_log`
- Format: plain text file
- Expected line order:
  - `timestamp: ...`
  - `user input: ...`
  - `assistant response: ...`

### Context Summary File Layout
- Report name: `context_summary`
- Format: plain text file
- Expected section order:
  - `Date: ...`
  - `Facts`
  - `Entities`
  - `Pending items`
  - `Decisions`
  - `Summary`

### Reminder JSON Layout
- Report name: `reminders`
- Format: JSON array
- Expected fields:
  - `id`
  - `title`
  - `datetime`
  - `status`
  - `completed_at` when applicable
- Field order:
  - not semantically required by JSON
  - new records are currently written in the order above
- Important notes:
  - generated structure must remain readable by current reminder service logic

### Working Context Layout
- Report name: `working_context`
- Format: markdown document
- Expected section order:
  - `# Working Context`
  - `## Automatic Context`
  - `### Last updated`
  - `### Current focus`
  - `### Active items`
  - `### Recent facts`
  - `### Relevant entities`
  - `### Open threads`
  - `## User Notes`
  - `### Stable preferences`
  - `### Personal context`
- Important notes:
  - automatic section bodies are regenerated by the application
  - user note sections are preserved during refreshes

### Active Context File Layout
- Report name: `active_context`
- Format: plain text file
- Expected section order:
  - `Recent entities`
  - `Active pending items`
  - `Recent topics`
  - `Recent decisions`

### Pending Items File Layout
- Report name: `pending_items`
- Format: plain text bullet list
- Expected line pattern:
  - `- <title> | <datetime>`
- Important notes:
  - when empty, the file currently uses `- None`

### CLI Context Output
- Report name: `context`
- Format: plain text markdown passthrough
- Important notes:
  - output mirrors `WORKING_CONTEXT.md`
  - headings and bullet lines should remain readable in the terminal

### Web Dashboard Layout
- Report name: `web_dashboard`
- Format: responsive browser page
- Expected sections:
  - hero header
  - conversation history with user prompts and assistant replies
  - single prompt composer area
- Important notes:
  - actions are driven by text prompts submitted with keyboard enter
  - the UI should not depend on action buttons for core flows
  - layout must remain usable on mobile widths
  - the page reads and writes through the HTTP API, not direct browser file access

### HTTP API Search Output
- Report name: `api_search`
- Format: JSON object
- Expected fields:
  - `query`
  - `logs`
  - `memory_map`

### HTTP API Overview Output
- Report name: `api_overview`
- Format: JSON object
- Expected fields:
  - `today_logs`
  - `recent_logs`
  - `pending_reminders`
  - `working_context`

### HTTP API Prompt Output
- Report name: `api_prompt`
- Format: JSON object
- Expected fields:
  - `mode`
  - `summary`
  - `payload`
