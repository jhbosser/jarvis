# AGENTS.md

## Project Objective
This repository contains a Python-based local assistant used for automation of personal information capture, data processing, reminder tracking, search, and simple report-style console outputs. AI agents must prioritize repository safety, preserve existing behavior, and improve the project through small, reversible documentation-first changes.

## Project Structure
- `jarvis.py`: CLI entrypoint for logging events, creating reminders, searching memory, and listing records for the current day.
- `reminder_service.py`: background service that checks pending reminders and triggers desktop notifications.
- `utils.py`: persistence helpers, file initialization, entity extraction, search, and memory map updates.
- `instructions.md`: operational rules for the assistant behavior already defined in the project.
- `memory_log.md`: append-only event log in plain text.
- `memory_map.md`: aggregated entity index derived from logged events.
- `reminders.json`: persisted reminder data and completion state.
- `docs/`: AI-oriented reference documentation for business rules, data contracts, report layouts, and validation.
- `examples/`: sample input/output artifacts that show expected data and layout formats.

## Key Files
- `PROJECT_CONTEXT.md`: high-level domain context that must be read before deeper changes.
- `README.md`: setup, execution flow, and operational overview.
- `docs/business_rules.md`: business and operational rules that must be preserved and updated when changed.
- `docs/data_contracts.md`: input and output data contracts, field expectations, and schema notes.
- `docs/report_layouts.md`: report and output layout reference.
- `docs/testing_checklist.md`: validation checklist to apply before finalizing changes.

## How to Run the Project
1. Create and activate a virtual environment.
2. Install dependencies from `requirements.txt`.
3. Run the CLI:
   - `python3 jarvis.py log "text"`
   - `python3 jarvis.py remind "title" "YYYY-MM-DD HH:MM"`
   - `python3 jarvis.py search "keyword"`
   - `python3 jarvis.py today`
4. Run the reminder service when needed:
   - `python3 reminder_service.py`

## Expected Outputs
- New log events appended to `memory_log.md`.
- Entity summaries refreshed in `memory_map.md`.
- Reminders persisted in `reminders.json`.
- Console tables and status messages displayed for search, today, and reminder flows.
- Desktop notifications generated by `reminder_service.py` when reminders become due.

## Development Rules
- Prefer small, localized changes over broad refactors.
- Preserve compatibility with existing input files and persisted storage formats.
- Do not alter column names or output layouts without explicit instruction.
- Do not infer fiscal, accounting, or financial rules.
- Never remove existing code without clear justification and user approval when impact is material.
- Preserve the structure of generated files and user memory records.
- Document assumptions instead of encoding speculative behavior.

## Safe Modification Rules
- Read `PROJECT_CONTEXT.md` before modifying code or documentation.
- Review `instructions.md` to preserve current assistant behavior.
- Update documentation before or alongside behavior changes.
- Do not modify source data directly.
- Scripts must not alter data in the source database.
- Do not rename persisted files (`memory_log.md`, `memory_map.md`, `reminders.json`) without explicit instruction.
- Avoid changing parsing formats for dates, logs, or reminders unless the user explicitly requests it.

## Data Sensitivity
- Treat logged memories and reminders as user data.
- Assume event logs may contain personal or operationally sensitive content.
- Avoid copying real user data into examples; use fictitious or reduced samples only.
- Prefer redacted examples when documenting inputs or outputs.

## Testing Guidelines
- Use `docs/testing_checklist.md` as the minimum validation baseline.
- Confirm the CLI still creates logs, reminders, searches, and daily views without errors.
- Confirm outputs remain compatible with the documented layouts and contracts.
- Validate that existing persisted files remain readable after changes.

## Documentation Rules

- Validations follow `docs/testing_checklist.md`.
- Report and output layouts live in `docs/report_layouts.md`.
